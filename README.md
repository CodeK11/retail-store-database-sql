# Retail Store Database Management using MySQL

A complete MySQL Retail Store Database project demonstrating database design, normalization, SQL queries, joins, aggregations, subqueries, and business reporting using a realistic retail sales dataset.


## Project Overview

This project demonstrates the design and implementation of a complete relational database for a retail store using MySQL.

The database models real-world retail operations including customers, products, orders, payments, order items, and customer reviews. It includes database creation, sample data insertion, and SQL queries that answer practical business questions.

This project showcases database design principles, SQL programming, and business analytics skills.

---

## Project Objectives

- Design a normalized retail database
- Implement relational tables with primary and foreign keys
- Populate tables with sample business data
- Answer business questions using SQL
- Demonstrate reporting and analytical querying

---

## Database Schema

The project consists of six relational tables:

- Customers
- Products
- Orders
- Order Items
- Payments
- Product Reviews

Relationships are enforced using Primary Keys and Foreign Keys to maintain data integrity.
